# 高频交易 (HFT) 技术内幕与延迟挑战

高频交易是量化领域中最依赖硬件和基础设施的“极限运动”。它的核心不是预测未来，而是**收割当下**。

## 1. 速度的量级 (The Scale of Speed)

| 动作 | 时间量级 | 说明 |
| :--- | :--- | :--- |
| 人类反应 | ~200 ms | 听到声音到按下的时间 |
| 网页请求 | ~50 ms | 浏览器打开一个页面的网络延迟 |
| LLM 决策 | 100ms - 2s | 处理复杂文本后的输出延迟 |
| **高频交易 (软件级)** | 1 - 5 μs | 基于优化过的 C++ 系统 |
| **高频交易 (硬件级)** | **< 100 ns** | 基于 **FPGA/ASIC** 芯片直接处理 |

## 2. 核心技术栈

### 2.1 编程语言
- **C++ (模板元编程)**：极度优化的执行流。
- **SystemVerilog/VHDL**：用于编写 FPGA 逻辑。

### 2.2 硬件加速
- **FPGA (Field-Programmable Gate Array)**：通过可编程硬件实现纳秒级处理，跳过 CPU 的指令循环。
- **Solarflare 网卡**：支持 OpenOnload 等内核旁路技术。

### 2.3 物理布局
- **Co-location (机房托管)**：服务器必须与交易所撮合引擎放在同一个机架或机房。

## 3. 常见高频策略
- **做市商策略 (Market Making)**：同时挂买单和卖单，赚取买卖价差 (Bid-Ask Spread)。
- **统计套利 (Arbitrage)**：捕捉不同市场间极短暂的价格不一致。
- **微观结构预测**：分析订单簿 (L2 Order Book) 的极速变动。

## 4. LLM 与 HFT 的对比

| 维度 | LLM 量化 | 高频交易 (HFT) |
| :--- | :--- | :--- |
| **主要目标** | 捕捉语义信号，预测中长线趋势 | 捕捉微观失衡，抢占成交先机 |
| **数据源** | 研报、财报、社交媒体、日线 | 全量 Tick 数据、L2 订单簿 |
| **核心瓶颈** | 模型推理时间 | 物理网络延迟、光速限制 |
| **硬件需求** | 大显存 GPU (H100/A100) | FPGA 显卡、超频 CPU、万兆网卡 |

## 5. 总结
高频交易是一场**“硬件战争”**，而 LLM 量化是一场**“逻辑战争”**。对于个人或非顶级机构，LLM 在处理非结构化信号上的优势更具有差异化竞争潜力。

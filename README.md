# AI 学习与知识沉淀库 (AI-Knowledge-Base)

> **自律、深度、系统化。** 这是一个致力于构建 AI 领域“第二大脑”的开源知识库项目，涵盖了从底层 Transformer 架构到最前沿的 DeepSeek 推理优化技术。

---

## 🌟 项目愿景
本项目不仅是笔记的堆叠，更是对 AI 知识颗粒度的极限追求。通过“AI 学习伴侣” (Gemini Agent) 的持续迭代，我们将碎片化的对话转化为结构化的本地 Markdown 知识库，旨在为 AI 研究者和开发者提供一份可搜索、可关联、深度的技术百科。

## 🏗️ 知识图谱架构

项目采用分层目录结构，确保知识的可扩展性：

### 1. [AI 核心领域](./AI/)
*   **[基础理论](./AI/基础理论/)**：Transformer 深度解析、KV Cache 机制、参数量与性能关系。
    *   **[推理机制](./AI/基础理论/推理机制/)**：**核心专题**。探讨分步推理、缓存局部性、分发路由。
    *   **[Prompt 工程](./AI/基础理论/Prompt工程/)**：Few-shot 技巧、缓存优化、Token 减支策略。
*   **[模型训练](./AI/模型训练/)**：微调技术 (SFT/LoRA)、模型蒸馏、多模态训练。
    *   **[DeepSeek 专题](./AI/模型训练/DeepSeek专题/)**：R1 推理能力、V4 序列压缩技术全解析。

*   **[语音技术](./AI/语音技术/)**：TTS、语音唤醒、VAD 算法。

### 2. [计算机基础](./计算机基础/)

## 🛠️ 管理与维护准则

本项目由 Gemini CLI Agent 协助维护，遵循以下严格准则（详见 [GEMINI.md](./GEMINI.md)）：
- **物理校验**：所有写入操作均执行读回校验，确保磁盘一致性。
- **详尽性优先**：文档豁免简洁性约束，追求技术细节的绝对颗粒度。
- **Gardening 机制**：定期进行目录重构，防止文件碎片化。

## 📖 如何使用
1.  **全局索引**：查看 [index.md](./index.md) 获取所有笔记的完整清单。
2.  **双向链接**：推荐使用 **Obsidian** 或支持 Wikilinks 的编辑器打开，体验网状知识连接。
3.  **实时更新**：笔记随学习进程动态演进，Update History 记录了每一次认知的提升。

# LLM 微调技术全量详解

本指南旨在为开发者提供大模型微调过程中涉及的每一项技术的深度拆解。涵盖了从显存管理的物理边界，到损失函数收敛的数学本质，旨在帮助解决“模型不收敛”或“准确率低下”等核心工程痛点。

---

## 1. LoRA (Low-Rank Adaptation) 核心参数的物理本质

LoRA 的核心思想是将参数更新 $\Delta W$ 分解为两个低秩矩阵的乘积，从而在冻结预训练权重的背景下实现高效微调。

### 1.1 LoRA 秩 (Rank, `r`)
- **物理意义**：定义了微调过程中允许模型更新的**特征维度空间**。它就像是模型学习新知识的“逻辑内存”。
- **深度影响分析**：
    - **逻辑拟合能力**：`r` 决定了模型能否捕捉到数据中复杂的非线性关联。如果 7000 条数据准确率卡在 75%，通常是因为 `r=8` 导致模型只能记住简单的模式（Style），而无法理解深层逻辑（Logic）。
    - **显存与参数量**：可训练参数量 $P \approx 2 \times L \times d \times r$（$L$ 为层数，$d$ 为隐藏层维度）。`r` 从 8 提升到 64，显存压力线性增加，但拟合上限呈指数级提升。
- **2026 建议**：指令遵循任务推荐 32-64；垂直领域深度知识注入（如法律、代码）强烈推荐 128+。

### 1.2 LoRA 缩放系数 (Alpha, `lora_alpha`)
- **物理意义**：控制 $\Delta W$ 对原始预训练权重 $W$ 的**干扰权重**。
- **数学本质**：$\text{Output} = Wx + \frac{\alpha}{r} \times (BA)x$。
- **作用分析**：设置 $\alpha$ 是为了在改变 `r` 时保持梯度的数值稳定性。通常建议 `alpha = 2 * r`。如果 $\alpha$ 过大，模型会迅速偏离预训练知识（变傻）；如果过小，微调几乎不生效。

### 1.3 目标模块 (Target Modules)
- **底层逻辑**：决定了“手术”动在模型大脑的哪个部分。
- **覆盖面差异**：
    - **q_proj / v_proj**：仅影响注意力注意力机制，适合极低显存场景。
    - **all-linear**（包括 `gate_proj`, `up_proj`, `down_proj` 等）：覆盖 MLP 模块。这是 2026 年的**工业标准**。
- **结论**：**必须选择 all-linear**。忽视 MLP 层会导致模型在知识记忆和事实性回答上表现极差，这是导致 75% 准确率瓶颈的最隐蔽原因。

## 2. 训练策略参数：决定“收敛路径”

### 2.1 学习率 (Learning Rate, `lr`)
- **物理意义**：优化器在负梯度方向上寻找最优解的**单步跨度**。
- **调优策略**：
    - **LoRA 模式**：由于只有极少参数在动，通常需要较大的 LR。推荐 `1e-4` 到 `2e-4`。
    - **全参数模式**：推荐 `1e-5` 到 `2e-5`。
- **风险提示**：如果 Loss 曲线震荡且无法降到 0.5 以下，优先检查学习率是否过高。

### 2.2 批处理大小 (Batch Size) 与 梯度累积 (Gradient Accumulation)
- **等效 Batch Size**：$\text{Batch Size} \times \text{Gradient Accumulation Steps}$。
- **物理意义**：决定了梯度估计的**平滑度**。
- **分析**：
    - 小等效 BS（<16）：梯度方向不稳定，容易在局部最优解反复横跳（这是 75% 准确率的常见诱因）。
    - 大等效 BS（32-128）：梯度更新稳健，有助于模型在大规模数据集上寻找全局最优解。

### 2.3 学习率调节器 (Scheduler: `cosine` vs `linear`)
- **`cosine` (余弦下降)**：在训练后期将步幅压缩到极小，有助于模型在最优解附近进行“精准降落”。
- **预热比率 (Warmup Ratio)**：建议设置在 0.05 - 0.1。它能防止模型在训练初期的巨大梯度冲力下直接崩溃（产生 NaN）。

## 3. 对齐与系统参数：决定“接头暗号”

### 3.1 对话模板 (Prompt Template)
- **物理意义**：定义了模型识别对话边界（System/User/Assistant）的控制字符（Control Tokens）。
- **致命陷阱**：如果你使用的是 Qwen 2.5，但选错了模板。
    - **后果**：模型会学习错误的起始符，导致推理时无法正确区分指令与背景，直接导致准确率在 70%-80% 处锁死。
- **结论**：必须严格对齐基座模型的官方模板。

### 3.2 截断长度 (Max Length)
- **物理意义**：定义了模型处理数据的**视野广度**。
- **风险分析**：如果设置过短（如 1024），但 7000 条数据中有大量长文本（如 2048），模型会因为看不到结尾而学到“残缺逻辑”，导致输出突然截断或复读。

### 3.3 计算精度 (`bf16` vs `fp16`)
- **建议**：在支持的硬件上（30/40 系, A100+）**强制使用 `bf16`**。它拥有更大的动态范围，能有效防止因溢出导致的 Loss 变 NaN。

## 4. 2026 高阶参数指南 (Advanced Tweaks)

### 4.1 NEFTune 噪声注入 (Noise Alpha)
- **作用**：通过在 Embedding 层加入噪声，防止模型过拟合到训练数据的字面表述上，显著提升泛化能力。

### 4.2 LoRA+ 差异化更新
- **作用**：将输出权重矩阵的学习率设为输入权重矩阵的 16 倍，大幅提升收敛速度并突破原本的精度瓶颈。

## 5. 针对 75% 准确率案例的参数一键对齐表

| 参数名称 | 调整方向 | 核心理由 |
| :--- | :--- | :--- |
| **LoRA Rank** | **8 -> 64** | 扩充逻辑内存，解决欠拟合 |
| **Target Modules**| **all-linear** | 激活模型全功能区，注入深度知识 |
| **Learning Rate** | **5e-5 -> 1e-4** | 给予 LoRA 足够的驱动力 |
| **Warmup Ratio** | **0 -> 0.1** | 稳定训练初期梯度 |
| **Gradient Acc** | **8 -> 16** | 提升梯度估计的准确性 (BS=32) |
| **Template** | **qwen** | 纠正模板对齐错误，彻底消除幻觉 |

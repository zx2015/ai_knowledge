# 微调泛化与死记硬背深度解析

当模型在微调后表现出“死记硬背”（换个问法就不会）和“幻觉”（胡言乱语）时，通常是由于模型从**语义理解**退化到了**样本记忆**。

## 1. 为什么会“死记硬背”？ (过拟合的根源)

### 1.1 数据多样性不足 (The Diversity Trap)
*   **现象**：如果你的训练数据中，问题的描述方式非常单一（例如全是“请解释：[术语]”）。
*   **原理**：模型非常“聪明”且“懒惰”。它会发现一个捷径：不需要理解词汇含义，只需要检测到“请解释”这个关键词，然后输出对应的文本片段即可。
*   **结果**：一旦你换成“[术语]是什么意思？”，模型因为没见过这种触发词，就无法激活正确的记忆路径。

### 1.2 训练轮数 (Epoch) 过多
*   **规律**：LLM 微调与传统机器学习不同。由于 LLM 参数量极大，它有极强的“背诵”能力。
*   **建议**：对于 SFT（指令微调），通常 **1-3 个 Epoch** 就足够了。超过 5 个 Epoch，模型往往会进入“逐字记忆”阶段，Loss 还在降，但泛化性大幅崩塌。

### 1.3 LoRA 参数配置不当
*   **秩 (Rank $r$) 过高**：如果 $r=128$ 甚至更高，LoRA 的参数量变大，其表达能力过强，容易捕获训练集中的噪声，导致过拟合。
*   **Alpha 与 Rank 的比例**：通常建议 $\text{Alpha} = 2 	imes \text{Rank}$。如果 Alpha 过大，更新步长相对过大，容易导致模型震荡或迅速收敛到局部最优（死记硬背点）。

## 2. 为什么换个问法就“幻觉”？

### 2.1 强行注入新知识 (Knowledge Injection vs. Style Alignment)
*   **核心痛点**：微调最擅长的是**对齐格式**（比如让模型学会用 JSON 输出），而不是**学习新知识**。
*   **原理**：如果微调数据中包含模型预训练时从未见过的私有事实，模型会尝试在参数中“硬刻”下这些事实。
*   **结果**：由于模型并没有真正理解这些新知识的逻辑，它会把这些新知识与旧知识混淆，产生看似合理但事实错误的“幻觉”。
*   **方案**：**RAG (检索增强生成)** 才是解决新知识注入的最佳方案，微调应只负责教模型如何使用 RAG 检索到的信息。

### 2.2 丧失了“拒绝回答”的能力
*   **原因**：如果训练集里全是“问题-正确答案”对，模型会认为“回答出东西”是唯一的奖励目标。
*   **后果**：当面对它不确定或换了问法的问题时，它不敢说“我不知道”，而是根据微调学到的片面关联，胡乱拼凑一个答案。

## 3. 解决策略：如何让模型变“聪明”？

### 3.1 增加 Prompt 多样性 (Data Augmentation)
*   **方法**：对同一条知识点，编写 5-10 种不同的问法。
    *   *原问法*：什么是 LoRA？
    *   *变体 1*：请用通俗的语言解释 LoRA 技术。
    *   *变体 2*：LoRA 的核心原理是什么？
    *   *变体 3*：对比一下全量微调和 LoRA。
*   **工具**：可以利用 GPT-4 或 Qwen 等大模型自动生成这些变体。

### 3.2 引入负面样本 (Negative Samples)
*   **策略**：在数据集中加入一些它不应该回答或需要拒绝回答的样本，并给它标记“对不起，根据目前的信息我无法回答”。这能显著降低幻觉。

### 3.3 混合通用语料 (Data Mixing)
*   **做法**：在你的垂直领域数据中，掺入 **5%-10% 的通用对话语料**（如 ShareGPT 数据集）。
*   **目的**：这就像在学专业课时顺便看点杂书，防止模型“学傻了”（灾难性遗忘）。

### 3.4 监控验证集 (Validation Monitoring)
*   **关键指标**：不要只看训练 Loss。要准备一个**与训练集问法完全不同**的验证集。如果训练 Loss 在降，但验证集 Loss 上升，立即停止训练。

## 4. 总结：避坑指南

| 维度 | 导致死记硬背的做法 | 促进泛化的做法 |
| :--- | :--- | :--- |
| **问法** | 固定模板，单一表达 | 丰富多样，涵盖各种语气 |
| **轮数** | 越多越好 (Epoch > 5) | 见好就收 (Epoch 1-3) |
| **目标** | 教它新知识 (事实性数据) | 教它新格式/逻辑 (指令对齐) |
| **参数** | 高 Rank LoRA ($r \ge 128$) | 适中 Rank ($r = 8, 16, 32$) |
| **成分** | 100% 垂直领域数据 | 垂直数据 + 部分通用数据 |

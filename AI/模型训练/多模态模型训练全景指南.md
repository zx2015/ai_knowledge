# 多模态模型训练全景指南

多模态大模型 (MLLM) 的目标是打破单一模态的限制，使模型能够像人类一样同时处理视觉、听觉和文本信息。其核心挑战在于**模态对齐 (Modality Alignment)**。

## 1. 核心架构设计

多模态架构通常由三个支柱组成：

### 1.1 模态编码器 (Modality Encoder)
- **视觉**：常用 CLIP-ViT (Vision Transformer) 或 ConvNeXt。
- **音频**：常用 Whisper。
- **特点**：通常使用预训练好的冻结权重，以利用其已有的特征提取能力。

### 1.2 模态连接器 (Connector)
- **线性投影层 (Linear Projector)**：如 LLaVA，使用简单的 MLP 将特征维度对齐。
- **可学习查询 (Query-based)**：如 BLIP-2 的 Q-Former，通过一组可学习的 Query 来提取视觉特征。
- **交叉注意力 (Cross-Attention)**：如 Flamingo，在 LLM 层间插入注意力机制。

### 1.3 强语言模型 (Backbone LLM)
- 如 LLaMA, Vicuna 或 Qwen。它是多模态指令理解和推理的核心。

## 2. 训练流程

### 阶段 1: 特征对齐 (Feature Alignment)
- **数据**：图文对 (Image-Text Pairs)，如 `{"image": "cat.jpg", "caption": "A fluffy cat sitting on a sofa"}`。
- **做法**：**冻结** Encoder 和 LLM，只更新 **Connector**。
- **意义**：让连接器学会将视觉信号转化为 LLM 熟悉的“视觉单词 (Visual Tokens)”。

### 阶段 2: 指令微调 (Visual Instruction Tuning)
- **数据**：包含对话、推理、OCR 的多轮对话数据。
- **做法**：同时更新 Connector 和 LLM (通常通过 LoRA 或全参数微调)。
- **意义**：赋予模型理解复杂指令和上下文的能力。

## 3. 数据处理流水线 (Data Pipeline)

### 3.1 为什么不直接使用 Base64 训练？
虽然在 API 调用或 JSON 数据集中常用 Base64 编码，但训练时**必须**将其解码。
- **效率低**：Base64 编码比二进制大 1/3，增加显存压力。
- **不可计算**：模型只能处理数值张量 (Tensors)，不能处理字符串。

### 3.2 图片的处理路径
1. **读取**：从磁盘读取 `.jpg` 或从 JSON 中解码 Base64。
2. **解码**：转化为 RGB 像素矩阵 (Raw Pixels)。
3. **变换 (Transforms)**：缩放 (如 224x224)、归一化 (Normalize)、中心裁剪。
4. **张量化**：转化为 `[C, H, W]` 的 Tensor 送入 GPU。

### 3.3 视频的处理路径
1. **抽帧 (Sampling)**：由于视频 Token 量巨大，通常从视频中均匀抽取 $N$ 帧 (如 8 帧或 16 帧)。
2. **空间特征**：每帧独立经过视觉编码器。
3. **时序建模**：加入 **Temporal Embedding** 或使用 **3D Convolution / Time-Attention** 来捕捉帧间的运动信息。

## 4. 关键对齐技术

### 3.1 对比学习 (Contrastive Learning)
代表作：**CLIP**。通过最大化配对图文的余弦相似度，最小化非配对的相似度。
- **Loss**: InfoNCE Loss。

### 3.2 预测掩码 (Masked Modeling)
代表作：**BEiT-3**。类似于 BERT，掩盖图像或文本的一部分并让模型预测缺失部分，强制模型学习跨模态关联。

### 3.3 生成式对齐 (Generative Alignment)
代表作：**LLaVA**。直接将视觉 Token 拼接到文本 Token 序列中，使用 Next Token Prediction 任务进行训练。

## 4. 常见挑战
1. **幻觉问题 (Hallucination)**：模型会生成与图像内容不符的描述。
2. **分辨率限制**：ViT 预训练分辨率通常较低 (224x224)，处理细粒度 OCR 任务较难。
3. **计算开销**：多模态 Token 通常很长（一个图像可能占用 256-576 个 Token），增加推理延迟。

# 语音唤醒技术 (KWS) 实现方案详解

语音唤醒（Keyword Spotting, KWS）是指让设备在低功耗状态下，通过实时监听音频流，精准识别出预设的特定词汇（如“小爱同学”、“嘿 Siri”）的技术。它是人机语音交互的第一入口。

## 1. 技术原理

语音唤醒的核心是一个**连续监听、低时延、低功耗**的二进制分类任务：判断当前的音频片段是“唤醒词”还是“非唤醒词”。

### 1.1 基本工作流
1.  **音频采集**：通常采用 16kHz 采样率、单声道音频。
2.  **前端处理**：包括降噪（ANS）、回声消除（AEC）和增益控制（AGC）。
3.  **特征提取**：将时域波形转化为频域特征，常用 **Fbank**（Mel 滤波器组）或 **MFCC**。
4.  **模型推理**：神经网络计算当前帧及前后上下文属于唤醒词的概率。
5.  **后处理（阈值决策）**：为了防止误触发，通常需要连续多帧的置信度超过设定阈值，并结合平滑算法输出最终结果。

## 2. 主流开源框架对比 (2025)

目前工业界最推荐的两个开源方案是 **WeKWS** 和 **Sherpa-ONNX**。

| 特性 | WeKWS (Webull) | Sherpa-ONNX |
| :--- | :--- | :--- |
| **定位** | 专为唤醒词设计的轻量级工具包 | 全能语音 AI 框架 (ASR/TTS/KWS/VAD) |
| **核心算法** | 深度可分离卷积 (DS-CNN)、TCN | Zipformer (Next-gen Kaldi) |
| **优势** | 生产就绪、模型极小、支持免对齐训练 | 跨平台能力极强、基于 ONNX Runtime |
| **支持语言** | 深度优化中文/英文 | 极其丰富（12 种编程语言接口） |
| **适用场景** | 物联网设备、专用唤醒模块 | 移动 App、桌面应用、综合性机器人 |

## 3. 实现步骤：以 Sherpa-ONNX 为例

实现一个基础的语音唤醒功能通常分为以下几步：

### 3.1 环境准备
```bash
pip install sherpa-onnx
```

### 3.2 获取预训练模型
从官方 Model Zoo 下载针对中文优化的轻量级模型（如 3.3M 参数版本）：
- `sherpa-onnx-kws-zipformer-wenetspeech-3.3M-2024-01-01`

### 3.3 核心代码逻辑 (Python)
```python
import sherpa_onnx

# 1. 配置模型参数
config = sherpa_onnx.KeywordSpotterConfig(
    feat_config=sherpa_onnx.FeatureConfig(sample_rate=16000, feature_dim=80),
    model_config=sherpa_onnx.OnlineModelConfig(model="model.onnx", num_threads=1),
    keywords_file="keywords.txt", # 存放唤醒词及其拼音/音素
    max_active_paths=4
)

# 2. 初始化唤醒器
kws = sherpa_onnx.KeywordSpotter(config)

# 3. 流式处理音频流
```

## 4. 硬件依赖与性能优化

### 4.1 CPU 运行能力
WeKWS 和 Sherpa-ONNX 的 KWS 模型**完全可以在 CPU 上高效运行**，并不依赖 GPU。
- **设计初衷**：为“始终在线（Always-on）”场景设计，强调低功耗和低延迟。
- **资源占用**：在现代 CPU（X86/ARM）上，CPU 占用率通常低于 5%。
- **硬件加速**：利用 CPU 的指令集加速（如 X86 的 AVX2/SSE, ARM 的 NEON）。

### 4.2 优化手段
1.  **INT8 量化**：将 FP32 模型量化为 INT8，可显著减少内存带宽占用，并在移动端提升 2-4 倍推理速度。
2.  **内存管理**：采用环形缓冲区（Ring Buffer）处理流式音频，确保内存占用恒定。
3.  **ONNX Runtime**：利用其跨平台推理引擎，自动匹配当前硬件的最佳执行提供者（Execution Provider）。

## 5. 关键难点与优化建议

1.  **误唤醒 (False Alarm) vs. 拒唤醒 (False Rejection)**：
    *   通过调整阈值（Threshold）平衡。
    *   增加**二次确认**机制：唤醒后开启更强大的 ASR 模型进行意图校验。
2.  **噪声抗性**：
    *   在训练数据中加入大量背景噪声（RIR 混响、白噪声、环境音）进行数据增强。
    *   前端使用双麦克风阵列进行波束成形（Beamforming）。
3.  **低功耗实现**：
    *   在嵌入式芯片上使用 **DSP** 或 **NPU** 进行计算。
    *   采用两级唤醒架构：低功耗简单模型初筛 + 高性能复杂模型精核。

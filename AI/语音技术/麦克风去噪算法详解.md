# 麦克风去噪算法详解

麦克风去噪（Noise Reduction, NR）是语音交互系统的核心预处理步骤。目前业界已全面转向“传统信号处理 + 深度学习”的混合架构。

## 1. 单麦克风去噪方案

### A. 传统 DSP 算法 (低功耗/高实时性)
| 算法 | 原理 | 特点 |
| :--- | :--- | :--- |
| **谱减法** | 减去估计的噪声频谱。 | 简单，但易产生“音乐噪声”。 |
| **维纳滤波** | 基于 SNR 调整频带增益。 | 效果平滑，常用于工程落地。 |
| **WebRTC NS** | 概率模型 + 维纳滤波。 | 开源标配，对平稳噪声（如风扇）效果好。 |

### B. 深度学习 AI 算法 (高性能/复杂环境)
- **RNNoise (Mozilla)**：经典信号处理 + GRU 神经网络。**极低延迟 (10-20ms)**，适合嵌入式。
- **DeepFilterNet**：两阶段增强模型。**当前 SOTA**（State-of-the-art），人声保真度极高。
- **Voice-Activity Detection (VAD)**：辅助算法，用于判断当前是否有真人说话，从而触发不同的降噪强度。

## 2. 多麦克风阵列方案 (空间过滤)

利用空间位置信息，实现精准的“声源定向”和“侧方干扰抑制”。

- **波束成形 (Beamforming)**：
    - **Delay-and-Sum**: 基础方案，增强特定角度。
    - **MVDR**: 智能抑制干扰方向，提高信噪比。
- **盲源分离 (BSS)**：在不知道声源位置的情况下，将混合在一起的多个声音分离开（如“鸡尾酒会效应”）。

## 3. 工业界“3A”音频处理套件
一个完整的麦克风处理流水线通常包含：
1. **AEC (Acoustic Echo Cancellation)**：回声消除（消除扬声器回授）。
2. **ANS (Automatic Noise Suppression)**：自动噪声抑制。
3. **AGC (Automatic Gain Control)**：自动增益控制（均衡音量）。

## 4. 推荐开源方案项目表

| 项目名称 | GitHub 仓库 | 核心优势 | 推荐语言 |
| :--- | :--- | :--- | :--- |
| **DeepFilterNet** | `Rikorose/DeepFilterNet` | 目前效果最自然、性能最强的 AI 降噪。 | Rust / Python |
| **RNNoise** | `xiph/rnnoise` | 极致轻量级，适合低性能嵌入式 CPU。 | C / C++ |
| **DTLN** | `breizhn/DTLN` | 竞赛级模型，对噪声鲁棒性极高。 | Python / TF |
| **WebRTC APM** | `get-wrecked/webrtc-audioprocessing` | 全套 3A (AEC/NS/AGC) 工程方案。 | C++ |

## 5. 快速上手建议
- **如果您想在 Python 中快速降噪**: 使用 `pip install noisereduce` (传统方案) 或 `pip install deepfilternet` (AI 方案)。
- **如果您是嵌入式开发 (C/C++)**: 优先集成 `rnnoise`，它没有复杂的依赖，编译非常简单。
- **如果您做 VoIP/音视频通话**: 建议直接使用 `webrtc-audioprocessing`，因为它处理回声消除 (AEC) 的能力是其他方案难以替代的。

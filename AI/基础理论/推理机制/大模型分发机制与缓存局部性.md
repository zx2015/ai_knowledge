# 大模型分发机制与缓存局部性

在大规模分布式推理集群中，如何确保相似请求被路由到相同的计算单元，是提升 **KV Cache 命中率** 的核心挑战。

## 1. 前缀感知路由 (Prefix-Aware Routing)
传统的负载均衡（如轮询）是“缓存盲视”的。高性能 Provider 通常采用前缀感知路由：
- **一致性哈希 (Consistent Hashing)**：对请求的 Prompt 前缀（Prefix）进行哈希，将请求映射到特定的计算节点。**注意：** 路由层并非对全量 Prompt 哈希，而是针对固定大小的“块（Blocks）”生成的哈希链。
- **分块哈希链 (Block-level Hash Chain)**：系统将 Prompt 切分为块（如 16/64 tokens），每块的哈希值依赖于前一块。只要前缀序列不变，其哈希指纹即保持稳定，从而避免了因 Prompt 末尾微小变动导致的“哈希雪崩”。

## 2. 状态追踪与 Radix 树
推理引擎（如 vLLM, SGLang）使用 **Radix Tree (基数树)** 管理本地缓存块：
- **Worker 状态报告**：计算节点定期向中心调度器同步其缓存的前缀指纹。
- **精确命中**：调度器根据各节点的缓存状态进行“得分制”分发，优先分发给前缀重合度最高的节点。

## 3. 全局共享缓存 (Distributed KV Cache)
当局部性路由失效时，通过分布式存储层进行补救：
- **存算分离 (PD Disaggregation)**：将 Prefill（计算密集）与 Decode（访存密集）解耦，KV Cache 存储在分布式文件系统（如 DeepSeek 的 **3FS**）中。
- **RDMA 传输**：不同节点间通过 RDMA 高速网络直接搬运 KV Cache，使得“逻辑上的全局缓存”成为可能，打破了物理节点的存储壁垒。

## 4. 优化收益
- **降低 TTFT (首字延迟)**：跳过已命中部分的计算，推理速度提升数倍。
- **提升吞吐量**：减少了重复的 Prefill 计算量，使得单张显卡能承载更多并发请求。
